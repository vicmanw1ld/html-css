LAS ETIQUETAS

Se escriben entre desigualdades, suelen ir por parejas y rodeando porciones de texto, Tienen una etiqueta de apertura y otra de cierre, la de apertura contiene al menos el nombre de la marca y el cierre comienza por "" Hay casos en los que no son necesarios apertura y cierre, por ejemplo:

  • imágenes
  • Saltos de linea
  • separadores
  • Se las conoce como marcas vacías img hr br etc.

Atributos

Los atributos proporcionan información adicional sobre la etiqueta, todas
las etiquetas pueden tener atributos (uno o más), siempre en la apertura, se
representan mediante clave="valor"

Tipos de atributos

Atributos básicos

Se pueden usar en todas las etiquetas

id = "texto" establece un identificador único a cada elemento dentro de una página web

class = "texto" establece la clase css que se aplica a los estilos de elementos

style = "texto estblece de forma directa los estilos css de un elemento

title = "texto"establece el título a un elemento; mejora la accesibilidad y los navegadores lo muestran
cuando el usuario pasa el ratón por encima del elemento

Atributos para internacionalización

Los usan las páginas que muestran sus contenidos en varios idiomas

lang = "código de idioma indica el idioma mediante un código predefinido
xml:lang = "código de idioma" indica el idioma del elemento mediante un código
dir = "/tr/rtl/auto"indica la dirección del texto

Atributos de eventos

Solo se usan en web dinámicas con javascript

Atributos de foco

Relacionados con la accesibilidad de los sitios web

Atributos globales

En HTML, los atributos globales son atributos comunes a todos los elementos HTML.
Se pueden utilizar para establecer propiedades o comportamientos de los elementos HTML,
independientemente del elemento específico en el que se utilicen.

La etiqueta Doctype

  • La declaración !Doctype debe ser la primera cosa en su documento html, antes de la etiqueta html
  • !doctype no es una etiqueta html es una instrucción para que el navegador web
    sepa en que versión de html está escrita la página
  • Hace referencia a una DTD porque se basaba en SGML.
    El DTD especifica las reglas para el lenguaje de marcado, de modo que los navegadores procesen el contenido
    correctamente. Ahora html no está basado en SGML por lo que no requiere una referencia a una DTD.

CABECERA head

Incluye toda la info y los recursos necesarios para generar la página
base define la URL usada en el navegador
meta representa los metadatos, como la descripción del documento, palabras clave, codificación de caracteres etc.
meta name="nombre de la propiedad" content = "valor de la propiedad"

EJEMPLOS

DEFINIR EL AUTOR
meta name"author" content="Juan Perez"
DEFINIR EL PROGRAMA CON EL QUE SE HA CREADO
meta name="generator" content="wordPress 2.8.5
DEFINIR LA CODIFICACIÓN DE CARACTERES
meta http-equiv="content-type" content="text/html; charset=UTF-8"
meta http-equiv="content-type" content="text/html; charset=UTF-8
Definir el copyright
meta name="copyright" content="librosweb.es"
Definir el comportamiento de los buscadore
meta name="robots" content="index, follow"
Definir las palabras clave del documento
meta name="keywords" content="diseño, css, hojas de estilos, web, html"
Definir una breve descripción
meta name = "description" content="Artículos de diseño web, usabilidad y accesibilidad"
atributos: name => para decribir el tipo de metadata
content => para especificar el valor
charset => para declarar el tipo de codificación de caracteres a utilizar para procesar el contenido. link => Este elemento especifica la relación entre el documento y un recurso externo
(generalmente usado para cargar archivos CSS).
El elemento puede incluir los atributos href para declarar la ubicación del recurso, rel para definir tip de relación
media para especificar el medio al que el recurso está asociado.
type y sizes para declarar el tipo de recurso y su tamaño
style para declarar estilos CSS dentro del documento. script para cargar o declarar código Javascript.
